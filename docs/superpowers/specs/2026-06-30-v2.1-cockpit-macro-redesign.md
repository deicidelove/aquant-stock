- 上游：[v2 重构总纲](2026-06-23-stock-system-v2-redesign.md)；v2 三板块已上线（main）。

## 背景

真实联调后用户反馈驾驶舱"信息单薄、对股民没用、不好看、内容大多是死的（盘中不跳动）"。根因：
1. 驾驶舱混入了选股列表（建仓名单/高分股），且只给 `代码/名称/综合分`，没回答"为什么/该不该动"——**定位错位 + 信息单薄**。
2. 大盘数字读 `daily_bar` 昨收，一天内是常数 → **盘中轮询也不动，感觉死的**。
3. 白底裸表格 → **缺盘面感**。

## 关键决策（已确认）

- **驾驶舱重新定位为纯宏观盘面**；选股内容（建仓名单/高分股）移到「辅助股民·选票」。
- 视觉走**深色量化驾驶舱**主题；列表加厚（端出 `decision` 已有的理由/作战计划/风险）。
- 盘中变化数据改读**快照表**，显示"更新于"，区分「盘中实时 / 收盘定」。

## 数据家底（交底，决定可做性）

| 模块 | 数据现状 | 处理 |
|---|---|---|
| 大盘指数 | `index_daily`（沪深300 收盘序列）✅ | 可加上证/创业板；用收盘序列画走势 |
| 市场情绪 | 涨跌家数/涨停/量能可算 ✅；**北向资金无源**（交易所 2024 起停实时北向）❌ | 情绪由 宽度+涨跌停+量能 合成；**北向剔除** |
| 大盘资金 | `fund_flow`（每只主力净额）✅ | 汇总成全市场净流入/流出 |
| 板块资金 | `sector_daily` 仅涨跌/家数，**无资金** ⚠️ | **新增 akshare 行业资金流源 + 表 + 刷新** |
| 异常资金 | 可派生 ✅（依赖 `fund_flow` 历史积累） | 今日主力净额 vs 近 N 日均值的 z-score 挑异常 |
| 盘中实时 | `quote_snapshot` 当前空（刷新未落数）⚠️ | 修刷新层：盘中真正落 quote/fund 快照 |

## 支柱① 驾驶舱 = 宏观五模块（后端 + 前端）

驾驶舱专做宏观，五模块（结论在上、可下钻到个股/板块成分）：

1. **大盘指数**：沪深300/上证/创业板 走势 + 今日涨跌幅。
2. **市场情绪**：涨跌家数、涨停/跌停数、量能（成交额）、**情绪温度**（综合打分：冰点↔过热，由宽度/涨跌停比/量能合成，给一句话解读）。
3. **大盘资金**：全市场主力净流入/流出总量（`fund_flow` 主力净额汇总）+ 近日趋势。
4. **板块资金**：各板块主力净额 排行 + 热力（红流入/绿流出，面积=|净额|）。需新增板块资金源。
5. **异常资金**：主力净额显著偏离常态（|z| ≥ 阈值，z = (今日净额 − 近N日均值)/近N日标准差）的板块/个股，给"今天钱往哪儿异动"。

选股内容从驾驶舱移除；驾驶舱任意标的可点击下钻到现有个股页（K线+研判）。

## 支柱② 深色主题 + 信息加厚

- **主题**：深色背景 + 琥珀/青强调、红涨绿跌、卡片/徽章/等宽数字/迷你走势。自建轻量 Tailwind UI 原子（`Card`/`Badge`/`Stat`/`SignalTag`/`Sparkline`），全站套用，告别白底裸表。
- **加厚**（主要落在「辅助股民·选票」，因选股移到那）：每只候选一张卡——信号徽章🟢买入/🟡持有/🔴回避 + 一句话理由 + 作战计划(买点/止损/目标/仓位) + 风险等级。数据来自现有 `decision`（离线，不联网）。

## 支柱③ 让数据活起来

- 修刷新层：交易时段真正落 `quote_snapshot`（现价/涨跌）与盘中 `fund_flow` 快照（确认 `refresh_quotes` 等任务在跑、写入成功）。
- 宏观模块的盘中数据改读快照表实时算（涨跌家数/指数/资金）；非交易时段回退收盘。
- 每个卡片标 **"更新于 HH:MM:SS"**，并标注「盘中实时 / 收盘定」徽章；前端盘中 TanStack 轮询即见跳动。

## 分期

- **Plan A 后端**：①新增板块资金流源 + 表 + 刷新；②市场情绪/大盘资金/异常资金 领域函数；③驾驶舱宏观端点（指数/情绪/大盘资金/板块资金/异常资金）；④盘中快照落库修复。pytest（fixture 注入 fund_flow/sector 资金，验证汇总/情绪分/异常 z-score）。

## 非目标（YAGNI）

- 不做北向资金（无可靠实时源）。
- 不引 UI 组件库（Tailwind 自建轻原子）；不做自定义换肤。
- 不重写量化/持仓功能逻辑，只换皮 + 接快照 + 加厚展示。
- 异常资金用简单 z-score，不做复杂统计模型。
