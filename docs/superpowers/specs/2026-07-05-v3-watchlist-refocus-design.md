# v3 产品转向 · 自选股看板（散户日常看盘）· 设计文档

- 日期：2026-07-05
- 状态：已确认（待写实现计划）
- 范围：产品重定位后的第一屏。先做「自选股看板」单屏 + 真实数据，量化收起。
- 上游：v2/v2.1 已上线但被判"失败产品"；本文件是转向依据。

## 背景与转向依据（调研结论）

真实使用后用户判"失败的产品":宏观/因子/回测这些是**量化/机构口径**,散户既不看也看不懂。调研主流 App:

- 东方财富 = 资金流向/龙虎榜/北向（"钱往哪走"）
- 同花顺 = 智能盯盘/盘面异动/涨停/**自选股预警**
- 雪球 = **自选股** + 社区讨论/资讯

**共性:都以「自选股/持仓」为中心屏,今日热点/资金为辅。** 而本项目根本没有自选股概念,首屏摆宏观指数/因子——**做给量化工程师,却用"散户有没有用"去量它**。这是失败根因:目标用户错位。

**决策:重定位为「给普通股民(含开发者自己)日常看盘」的产品。** 以自选/持仓为核心屏;量化/因子/回测降级为"高级"入口。

## 本期范围（只做一屏,做深做透）

不再一次铺多板块。**只做「自选股看板」单屏 + 把它的真实数据灌满**。若这一屏每天愿意打开,方向即被验证,再外扩(今日热点等);否则成本小、快速证伪。

## 目标用户与成功标准

- 用户:普通股民 / 开发者本人,盘后或盘中打开看"我关注的票现在怎样、要不要动"。
- 成功标准:看板里每只票都有**真实、非空**的内容(行情+迷你K线+一句话信号+关键价位+提醒),且用通俗语言直接给"结论",不用自己扫数字。

## 数据可行性（关键:看板不会空）

看板每只票所需,**绝大部分现有 `daily_bar` 全市场日线即可算出**,不依赖尚未灌的快照/因子历史:

| 看板字段 | 来源 | 现状 |
|---|---|---|
| 名称 | `stock_basic` | ✅ |
| 现价/涨跌 | 盘中 `quote_snapshot`(有则用) 回退 `daily_bar` 最新收盘 | ✅ 回退可用 |
| 迷你 K 线 | `daily_bar` 最近 N 日 | ✅ |
| 一句话信号+关键价位(买点/止损/目标)+风险 | `research.decision(code, offline=True)` | ✅ 离线、只读库 |
| 异动提醒(跌破止损/到压力/信号转空) | `portfolio.holdings.sell_alerts` 复用 | ✅ |
| 最近一条消息 | `news_cache`(有则显示) | ⚠️ 需对自选票预取,缺则留空不报错 |

结论:**看板对任何真实股票都能立刻出内容**(信号/价位/K线来自已有日线),news 缺失优雅降级。这正好治"空/死"。

## 第 1 节 · 后端

**新增 `watchlist` 表**（`TABLE_KEYS=["code"]`）：`code`、`added_ts`。用户加/删自选。

**看板服务**（`aquant/portfolio/watchlist.py` 或复用 holdings 模块，纯领域、只读+对 watchlist 写）：
- `add(code)` / `remove(code)` / `list_codes()`
- `board() -> list[dict]`：对 **自选 ∪ 持仓** 去重后的每只 code，组装 `{code,name,last_price,pct_chg,kline:[...],signal,one_liner,battle_plan,risk_level,alerts,news?}`。`decision(offline=True)` 取信号/价位；`sell_alerts` 取提醒；`daily_bar` 尾段取 kline；`news_cache` 取一条。

**端点**：
- `POST /api/watchlist {code}` / `DELETE /api/watchlist/{code}` / `GET /api/watchlist`（自选码列表）
- `GET /api/board`（看板：自选∪持仓的完整卡片数据）

**数据灌入**：本期看板数据来自已有全市场 `daily_bar`（decision/kline 即时算）；无需先跑重刷新。news 预取沿用既有 `research_cache`/`news_cache`（可选，对自选票批量预取一次；缺失不阻塞）。

## 第 2 节 · 前端

- **导航重构**：默认首页 = **看板** `/`；保留 我的持仓、（今日热点占位/后续）；把 驾驶舱(宏观)/选票/复盘/量化回测/因子 收进"**高级**"下拉或次级入口（不删,降级）。
- **看板页** `/`：顶部一条大盘脸色（复用 sentiment 一句话，压缩成 strip）；下面自选/持仓卡片列表——每卡:名称+现价涨跌、迷你 K 线（ECharts sparkline）、信号徽章+一句话理由、买点/止损/目标、⚠️提醒、（可选）最近消息；卡片可点进个股详情（现有 `/stock/:code`）。
- **加自选**：顶部输入框输代码 → `POST /api/watchlist` → 刷新看板（mutation+invalidate）。
- 深色主题沿用 v2.1 原子。

## 第 3 节 · 分期

- **Plan A 后端**：`watchlist` 表 + 看板服务(add/remove/list/board) + 4 端点。pytest（seed 注入 daily_bar/持仓，验证 board 组装/自选∪持仓去重/加删）。
- **Plan B 前端**：导航重构（量化降级）+ 看板页 + 加自选 mutation + 迷你K线。Vitest。

## 非目标（YAGNI）

- 本期不做「今日热点」屏（涨停/龙虎榜/板块资金）——占位/后续。
- 不删除量化/驾驶舱/复盘功能,只在导航降级。
- 不做北向、不做社区/讨论、不做实时 tick。
- news 预取尽力而为,缺失降级,不为它阻塞看板。

## 未决 / 计划阶段再定

- 看板卡片迷你 K 线的根数（如 30/60）。
- 自选∪持仓的排序（涨跌幅?综合分?自选在前?）。
- "高级"入口的具体形态（下拉 vs 独立次级页）。
- news 是否本期就对自选票批量预取（可延后）。
