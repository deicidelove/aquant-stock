# v3.3 龙虎榜（游资 / 机构席位）

## 背景与动机
资深股民看盘的核心信息之一是**龙虎榜**：谁在买、谁在卖、机构还是游资、净买多少。
当前系统完全没有龙虎榜。akshare 数据已验证可用：
- 个股级 `stock_lhb_detail_em(start,end)` → 上榜列表（code/name/date/reason/close/pct_chg/lhb_net_buy/lhb_amount），单次网络调用。
- 席位级 `stock_lhb_stock_detail_em(code,date,flag)` → 买/卖前五营业部（营业部名称/买入/卖出/净额/类型）。

## 目标
1. **今日龙虎榜总览**：最近一个交易日上榜个股，按净买额排序；标注上榜原因、涨跌幅、机构/游资参与标签。
2. **个股席位穿透**：任一上榜个股 → 买卖前五营业部、净额，识别 机构专用 / 北向(深沪股通) / 知名游资 / 普通营业部。
3. **席位分类器**：把原始营业部名称归类成 机构 / 北向 / 游资 / 普通，并对已知知名游资打标（章盟主、赵老哥、量化席位等，可扩展词典）。

## 铁律遵守
- API/domain 只读 DuckDB，请求路径不发起第三方网络。
- 仅 refresh 异步任务发起网络：EOD 任务抓当日上榜列表 + 逐只抓席位入库。
- 席位逐只抓取（今日上榜 N 只 × 买卖两向 ≈ 2N 次调用）放在 EOD 异步任务，非请求路径。

## 数据模型（DuckDB 新表）
- `lhb_detail`（已在 TABLE_KEYS：code,date,reason）：上榜个股明细。
  列：code,name,date,reason,close,pct_chg,lhb_net_buy,lhb_amount
- `lhb_seat`（新增 TABLE_KEYS：code,date,side,seat）：席位明细。
  列：code,date,side('buy'/'sell'),rank,seat(营业部名),buy,sell,net,seat_type(inst/north/hotmoney/normal),hotmoney_name(可空)

## 域层 `aquant/lhb.py`（只读、offline）
- `lhb_today(limit=50)` → {date, rows:[{code,name,pct_chg,lhb_net_buy,lhb_amount,reason,tags:[...]}] }
  - tags 由 reason + 该股席位聚合推断（有机构专用→"机构"，有北向→"北向"，有已知游资→游资名）。
  - 按 lhb_net_buy 降序。
- `lhb_stock(code, date=None)` → {code,name,date,reason,buy:[seat...],sell:[seat...]}
  - date 缺省取该股最近上榜日。每个 seat：{rank,seat,buy,sell,net,seat_type,hotmoney_name}。
- 席位分类 `classify_seat(name)`：
  - 含"机构专用" → inst
  - 含"股通"（沪股通/深股通专用） → north
  - 命中 HOTMONEY 词典关键字 → hotmoney（返回别名）
  - 否则 normal

## refresh 任务 `server/refresh/lhb.py`
- `refresh_lhb(days=1)`：抓最近 days 个交易日上榜列表 save 到 lhb_detail；对最新交易日每只个股抓买/卖席位 save 到 lhb_seat。
- 挂到 scheduler EOD job（15:30 之后）。

## API（新 router `server/routers/lhb.py`，prefix /api/lhb）
- `GET /api/lhb/today?limit=50` → LhbTodayResp
- `GET /api/lhb/stock/{code}` → LhbStockResp（个股席位穿透）

## 前端
- 新页 `web/src/pages/Lhb.tsx`：今日龙虎榜表格（代码/名称/涨跌幅红绿/净买额/原因/标签badge），行可点开席位。
- 席位区块组件 `LhbSeats`：买方绿列 / 卖方红列，营业部+净额，类型badge（机构=蓝/北向=青/游资=橙/普通=灰）。
- 导航加入口；个股详情页 StockDetail 底部追加"该股龙虎榜席位"（若有）。
- types/client/hooks 对齐。

## 拆分
- **B1 后端**：source 席位函数 + store TABLE_KEYS + lhb.py 域层 + refresh 任务 + router + schemas + 测试。
- **B2 前端**：types/client/hooks + Lhb 页 + LhbSeats 组件 + 导航 + StockDetail 追加 + 测试。

## 数据缺口 / 取舍
- 游资识别依赖营业部名词典，先内置一小份知名游资 + 机构/北向规则；词典可后续扩充。
- 席位仅前五（akshare 限制），够用。
- 历史回补：首次可 refresh_lhb(days=5) 播种最近一周。
