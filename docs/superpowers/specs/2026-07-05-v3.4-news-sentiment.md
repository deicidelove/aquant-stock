# v3.4 市场消息面 / 新闻情绪

## 背景
用户反馈：「有一部分市场情绪是根据新闻来的，没有体现」。
现状：个股级新闻已有（`news.py:stock_news` + research 关键词抽取利好/利空进研判报告）。
缺口：**市场级消息面**——大盘财经快讯 + 聚合情绪，前端完全没有体现。

## 数据（已验证）
- `ak.stock_info_global_em()` → 200 条全球财经快讯，列 标题/摘要/发布时间/链接。可靠。

## 目标
1. 市场新闻流：抓全球财经快讯，落库缓存。
2. 新闻情绪打分：关键词法给每条打 利好(+1)/利空(-1)/中性(0)，聚合成 0-100 情绪指数。
3. 前端「📰 消息面」区块：情绪指数(gauge/数字) + 最新快讯列表(利好红/利空绿/中性灰标签)，放看板 + 驾驶舱。

## 铁律
- API/domain 只读 DuckDB，请求路径不发网络。
- 仅 refresh 任务发网络（抓快讯+打分入库），挂盘中 job（新闻高频，upsert 去重）。

## 域层 `aquant/sentiment.py`
- `POS`/`NEG` 关键词表（扩展 research.py 现有表，覆盖宏观：降准/降息/政策利好/外资流入 vs 加息/监管/违约/风险）。
- `score_text(text) -> int`（+1/-1/0，命中利好优先，再利空，否则中性）。
- `aggregate(items) -> {score,label,pos,neg,neutral}`：score = round(50 + 50*(pos-neg)/max(pos+neg,1))，clamp 0-100；label 由 score 分档(极度乐观/乐观/中性/谨慎/悲观)。
- `market_news_sentiment(limit=30) -> {score,label,pos,neg,neutral,items:[{title,time,url,sent}]}`：读 market_news 表(offline)，聚合。无表返回空结构。

## source `aquant/data/sources/news.py`
- `market_news(limit=50) -> list[dict]`：包 `stock_info_global_em`，rename 标题→title/摘要→summary/发布时间→time/链接→url，head(limit)。

## store
- 新表 `market_news`，键 `[time, title]`。列：time,title,summary,url,sent(int)。

## refresh `server/refresh/news.py`
- `refresh_market_news(limit=50)`：抓 market_news，逐条 score_text 写 sent，save。挂 intraday job。

## API（macro router，属大盘/驾驶舱）
- `GET /api/macro/news-sentiment?limit=30` → NewsSentimentResp{score,label,pos,neg,neutral,items}。

## 前端
- 组件 `web/src/components/NewsSentiment.tsx`：顶部情绪指数条(0-100，红绿色阶)+ 利好/利空/中性计数；下面快讯列表(时间+标题+情绪色点，链接可点)。
- 看板 Board 顶部加紧凑版(情绪数字+前 3 条)；驾驶舱 Cockpit 加完整版。
- types/client/hooks 对齐。

## 拆分
- C1 后端：sentiment 模块 + source market_news + store 键 + refresh + API + 测试。
- C2 前端：types/client/hooks + NewsSentiment 组件 + Board/Cockpit 接入 + 测试。

## 取舍
- 关键词法情绪，非 LLM/模型（离线、零依赖、可解释）；词典可后续扩充。
- 快讯为全市场，不做个股映射（个股新闻已在研判报告体现）。
