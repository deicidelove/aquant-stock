# v3.2 专业个股 K 线 · 设计文档

- 日期：2026-07-05
- 状态：已确认（待写实现计划）
- 范围：把个股详情页 `/stock/:code` 的"裸蜡烛"升级为专业三联图看盘。
- 上游：反馈"K线不像K线、不专业"；调研得专业看盘图 = 主图(K线+均线) + 副图(成交量 + MACD)。

## 背景

现 `/stock/:code` 的 K 线只有裸蜡烛(`buildKlineOption`)，无均线、无副图，不符合资深股民的看盘习惯。专业软件(同花顺/通达信)标准：主图 K线+MA，副图成交量 VOL + MACD。个股 `daily_bar` 已含 OHLC+volume，均线/MACD 可算——**能立刻做专业**。

## 关键决策（已确认）

- 三联图（共享 x 轴 + 十字光标 + 缩放）：
  1. **主图**：K线蜡烛(红涨绿跌) + 均线 **MA5/MA10/MA20/MA60** + 可选关键价位标注(止损/目标,来自本页已有 decision)
  2. **副图① 成交量**：量柱(随当日涨跌红/绿)
  3. **副图② MACD**：DIF 线 / DEA 线 / 红绿柱(hist)
- 交互：十字光标三图联动、滚轮/滑块缩放、tooltip 显示 OHLC+量+MACD
- 周期：日 K（现有数据）；分时/周月 K 后续（不在本期）
- 指标范围：MA + VOL + MACD（KDJ 等本期不做，YAGNI）

## 第 1 节 · 后端

`aquant/research.py` 或新 `aquant/chart.py` 新增纯领域函数（只读库）：
- `stock_chart(code: str, n: int = 250) -> dict`：读 `daily_bar` 尾 n 行，算 MA5/10/20/60、MACD(DIF/DEA/hist，EMA12/26/9)，返回：
  ```
  {code, bars:[{date,open,high,low,close,volume}],
   ma:{ma5:[..],ma10:[..],ma20:[..],ma60:[..]},        # 与 bars 对齐，前段不足为 None
   macd:{dif:[..],dea:[..],hist:[..]}}
  ```
  MACD 用标准 EMA(12)/EMA(26)→DIF，DIF 的 EMA(9)→DEA，hist=(DIF−DEA)*2。
- 复用：MA=rolling mean；MACD 可复用/参考 `signal/timing.py` 的 EMA 写法（现有 macd_signal 用 ewm span12/26/9）。

端点：`GET /api/stock/{code}/chart?n=250` → 上述结构（无数据 404 或空 bars）。挂在既有 `server/routers/stock.py`。

保留既有 `/kline`（board 迷你线仍用）不动。

## 第 2 节 · 前端

- 新增图表 option 纯函数 `buildProKlineOption(chart, {stop?, target?})`（`charts/options.ts`）：ECharts 3 grid（主图 60% / 量 18% / MACD 22%），`axisPointer` link 三图十字光标，`dataZoom`(inside+slider) 横跨三图；主图 candlestick + 4 条 MA line；量图 bar(涨红跌绿)；MACD dif/dea line + hist bar(正红负绿)；可选 markLine(止损/目标)。
- 新 client `getStockChart(code,n)` + hook `useStockChart(code,n)`（`enabled:!!code`，无需盘中轮询）。
- 新组件 `ProKlineChart({ chart, plan? })`（用 EChart 薄壳 + buildProKlineOption；plan 来自本页 report 的 battle_plan）。
- `StockDetail` 用 `ProKlineChart` 替换现有 `buildKlineOption` 的裸蜡烛区（保留研判 ReportPanel）。

## 第 3 节 · 分期

- **Plan A 后端**：`stock_chart` 领域函数(MA/MACD) + `/api/stock/{code}/chart` 端点。pytest（fixture daily_bar 验证 bars/MA 尾值/MACD 结构）。
- **Plan B 前端**：`buildProKlineOption` + client/hook + `ProKlineChart` + 接入 StockDetail。Vitest。

## 非目标（YAGNI）

- 不做分时图、周/月 K（需分钟/重采样，后续）。
- 不做 KDJ/BOLL 等更多副图指标（先 MA+VOL+MACD）。
- 不做多周期切换 UI、不做画线工具。
- 指数 K 线不在本期（指数无 OHLC，另需补数据）。
