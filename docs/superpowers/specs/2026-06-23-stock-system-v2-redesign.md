# A股量化投研系统 v2 重构 · 设计文档

- 日期：2026-06-23
- 状态：已确认（待写实现计划）
- 范围：系统级重构总纲。三大业务板块 + 数据时效地基，分期实现，每期各自 spec→plan→实现。

## 背景与动机

现状问题（用户反馈）：

1. **数据时效边界不清**：哪些实时、哪些异步没有分层。页面渲染时掺杂实时第三方联网调用，导致卡顿与脆弱（典型：聚焦个股因东财筹码接口断流，`_robust` 重试叠加退避拖满 ~27s）。
2. **业务组织混乱**：11 个并列页面（驾驶舱/大盘/决策/推荐跟踪/模拟盘/网格/个股/选股/资金流/板块/概览）堆在一个 700 行的 Streamlit `app.py` 里，按"功能碎片"排列而非"用户目的"。
3. **性能/体验有结构性天花板**：Streamlit 每次交互整脚本重跑，再优化也有限。

目标：数据时效分层 + 业务按用户目的重组为三大板块 + 性能体验质的飞跃。

## 关键决策（已与用户确认）

| 维度 | 决策 |
|---|---|
| 实时粒度 | 盘中准实时（分钟级快照），非 tick 级 |
| 前端形态 | 后端 API + 现代前端（替代 Streamlit） |
| 技术栈 | FastAPI + DuckDB + 后台刷新层；React+TS+Vite + ECharts + Tailwind/shadcn + TanStack Query |
| 落地顺序 | 阶段1 地基+驾驶舱走通全栈 → 阶段2 辅助股民 → 阶段3 量化 |

## 第 1 节 · 数据时效架构（地基）

**核心铁律：前端/API 永不直接联网取第三方数据。所有第三方数据由「后台刷新层」落库，API 只读 DuckDB。** 这条铁律从结构上消灭"渲染时联网阻塞"这一类问题。

数据按"刷新驱动来源"分三档：

| 数据 | 时效档 | 刷新驱动 | 落库表 |
|---|---|---|---|
| 日线 OHLC | 🔵批量 T+0 | 收盘后 run-daily | `daily_bar` |
| 因子值 / 综合评分 | 🔵批量 T+0 | 收盘后**物化** | 🆕`factor_score`（查表 ms 级，替代每次现算 30-60s） |
| 推荐 / 择时 / 模拟净值 | 🔵批量 T+0 | 收盘后 | `picks_log` / `paper_nav` |
| 大盘指数 / 板块涨跌·资金 | 🟡盘中快照 | 交易时段每 1-3min | 🆕`index_snapshot` / `sector_snapshot` |
| 个股现价 / 涨跌幅 | 🟡盘中快照 | 交易时段每 1-3min | 🆕`quote_snapshot` |
| 基本面 PE/PB/财务 | 🟢异步低频 | 每日夜间 | `fundamental` |
| 筹码分布 / 新闻 | 🟢异步预取 | 后台预取热门股 + 可降级 | 🆕`chip_cache` / `news_cache` |

**后台刷新层**（APScheduler，三类定时任务）：

- 盘中高频（9:30–11:30、13:00–15:00，每 1–3min）：`quote_snapshot`、`sector_snapshot`、`index_snapshot`、资金流快照。
- 收盘后（~15:30）：跑批 run-daily（日线增量、因子计算、综合评分**物化**、推荐、模拟净值）。
- 夜间低频（每日/每周）：补 `fundamental`，预取热门股 `chip_cache`/`news_cache`。

容错：东财断流等只影响后台任务（自带熔断/降级，复用现有 `_robust` 交替代理重试），**绝不阻塞用户页面**。盘中快照任务幂等、可补偿。

## 第 2 节 · 系统架构

```
                 ┌─────────────────────────┐
  第三方数据源 ──>│  后台刷新层 (APScheduler) │──写──┐
 (akshare/东财/新浪)└─────────────────────────┘      ▼
                                              ┌──────────────┐
                                              │   DuckDB     │
                                              └──────────────┘
                                                     ▲ 只读
   aquant/ (领域核心:数据/因子/策略/回测,复用) ───────┤
                                              ┌──────────────┐
                                              │  FastAPI     │ 出 JSON
                                              └──────────────┘
                                                     ▲ HTTP
                                              ┌──────────────┐
                                              │ React SPA    │ ECharts/局部刷新
                                              └──────────────┘
```

目录结构：

```
stock/
  aquant/        # 现有领域核心，保留并复用（数据/因子/scorer/timing/backtest/paper）
  server/        # 🆕 FastAPI：routers/ schemas/ deps.py refresh/(刷新层任务)
  web/           # 🆕 React+TS+Vite：components/ pages/ charts/ api/(TanStack Query)
  data_store/    # DuckDB 不变
```

- `aquant/` 作为领域核心被 FastAPI 直接 import 复用（不重写策略/回测逻辑）。
- 旧 Streamlit `app.py` 在迁移完成后归档退役。
- 单用户本地部署（macOS）。FastAPI 与刷新层可同进程（APScheduler 内嵌）或拆独立 worker，阶段1 先同进程。

## 第 3 节 · 三大业务板块（按"用户目的"重组）

### 板块① 驾驶舱（总览 → 下钻）

吸收现 cockpit / 大盘 / 板块 / 资金流 / 推荐跟踪 / 概览。

- **总**：大盘走势（指数 K + 攻防 regime） ｜ 板块概览（热力树图） ｜ 各板块最近一日资金流向 ｜ 市场情绪（涨跌家数 / 北向 / 情绪指标） ｜ 量化交易一览（因子 IC / 策略状态） ｜ 推荐收益一览（记分卡）。
- **分（下钻）**：各板块 Top-N 个股 K 图 → 高分股 K 图 → 单股研判报告（决策仪表盘）。

### 板块② 辅助股民（闭环：选票→买卖时机→收益→优化→再选）

吸收现 决策 / 选股 / 个股 / 推荐跟踪 + 🆕我的持仓。

- 选票（多因子打分 + 每日建仓名单 Top3） → 买卖时机（择时信号 + 关键价位：支撑/压力/止损） → 收益（推荐跟踪 + 持仓盈亏） → 优化（命中率/超额/调参建议） → 循环持续迭代。

### 板块③ 量化（闭环：金额→因子→模拟→收益→优化因子→选股模拟→收益分析→再优化）

吸收现 模拟盘 / 网格 / 回测 / 因子研究。

- 设初始金额 → 选因子/权重（IC 加权 vs 动量风格） → 模拟实战（paper/回测） → 收益分析（净值/绩效/归因） → 优化因子（IC 评估、权重对比） → 选股模拟实战 → 收益分析 → 循环。

## 第 4 节 · 性能体验跃迁点

1. 渲染只读库 → 全站秒开、0 联网阻塞（从结构上消灭 27s 类问题）。
2. 物化因子评分表 → 全市场打分 30–60s → 查表 ms 级。
3. 组件级局部刷新 / 下钻，不再整页重跑。
4. TanStack Query 服务端缓存 + 盘中自动轮询（交易时段按 refetchInterval；收盘后停轮询）。
5. 骨架屏渐进加载 + ECharts 流畅 K 线交互（缩放/十字光标）。
6. 后台预取热门股筹码/新闻 → 详情页秒开。

## 落地分期

每期各自 spec → plan → 实现，可独立交付。

- **阶段 1（下一步细化实现计划）**：地基（后台刷新层 + 物化评分表 + FastAPI 骨架 + 前端壳 + TanStack 接线） + **板块① 驾驶舱**，走通全栈。
- **阶段 2**：板块② 辅助股民。
- **阶段 3**：板块③ 量化。

## 非目标（YAGNI）

- 不做 tick 级实时 / WebSocket 流式推送。
- 不做多用户 / 鉴权 / 云部署（单用户本地）。
- 不重写 `aquant/` 领域核心（数据/因子/策略/回测复用）。
- 阶段 1 不并行铺开三板块，仅驾驶舱走通。

## 未决 / 阶段1 计划阶段再定

- 物化评分表的具体 schema 与重算触发点。
- 盘中快照的精确间隔（1 / 2 / 3min）与"热门股"预取清单的定义口径。
- FastAPI 与刷新层是否拆进程（阶段1 默认同进程）。
